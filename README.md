# Создание интерактивного дашборда в Yandex DataLens для анализа эффективности продаж косметических брендов

[![DataLens](https://img.shields.io/badge/DataLens-View_Dashboard-blue?style=for-the-badge&logo=yandex&logoColor=white)](https://datalens.yandex/0jexmaa3zg10k)

Интерактивный BI-дашборд, разработанный для анализа продаж, выручки и ключевых продуктовых метрик косметических брендов. Проект выполнен по методологии сквозного проектирования: от сбора бизнес-требований на основе интервью до создания интерактивного прототипа.

---

## 📌 Бизнес-контекст и задачи
В рамках проекта была реализована роль **ведущего аналитика**, взаимодействующего с **руководителем продуктовой команды** группы компаний в сфере красоты. 

**Основная цель:** спроектировать и разработать инструмент для категорийных менеджеров, бренд-менеджеров, маркетологов и аналитиков, позволяющий оперативно отслеживать ключевые метрики без ежемесячного ручного сбора данных из разных источников, а также презентовать результаты на встречах с CEO в Zoom.

### Бизнес-задачи проекта:
1. **Анализ эффективности текущего ассортимента:** выявление наиболее прибыльных брендов и категорий, а также оценка динамики по месяцам.
2. **Управление взаимоотношениями с брендами:** принятие решений о привлечении новых брендов, поддержке существующих или отказе от сотрудничества (если бренд стабильно показывает низкие продажи и выкуп).
3. **Оптимизация ассортимента:** выявление позиций с низким спросом для их последующего вывода или стимулирования спроса (акции, скидки, кросс-продажи, пакетные предложения).
4. **Анализ зависимости выкупа от стоимости:** исследование связи процента выкупа и цены товаров для оптимизации скидочных предложений на дорогие позиции.

---

## 🎯 Ключевые метрики и целевые ориентиры (KPI)
Для оценки успешности бизнеса и своевременного реагирования на изменения рынка на дашборде зафиксированы следующие целевые ориентиры:
* **Общая выручка** — целевой ориентир: рост не менее **5% месяц к месяцу (MoM)** как в целом по компании, так и по отдельным брендам/категориям.
* **Продажи** (в штуках, с учетом отказов).
* **Средняя цена товара**.
* **Процент выкупа товаров** (выкупленные товары / заказы) — целевой ориентир: достижение планового показателя в **95%**. Любое отклонение вниз является триггером для детального анализа.

---

## 🧠 Логика принятия бизнес-решений на основе данных
Дашборд выступает основным инструментом принятия решений во время ежемесячных Zoom-встреч с CEO:
* **При невыполнении плана по выручке (+5% MoM):** выявляются проблемные сегменты (категории или бренды с наибольшим падением), для которых разрабатываются маркетинговые меры (запуск акций, промо-кодов, скидок и кросс-продаж).
* **При низком проценте выкупа дорогих товаров:** разрабатываются точечные скидочные предложения для снижения барьера покупки и повышения спроса.
* **При систематически низких показателях бренда:** если бренд стабильно не демонстрирует роста продаж и не достигает целевого выкупа, принимается решение о прекращении сотрудничества.

## 🛠️ Стек и инструменты
* **BI-система:** Yandex DataLens
* **Проектирование требований:** фреймворк Dashboard Canvas (подготовка и проведение интервью, определение пользователей, контекста использования, макетирование)
* **Источники данных:** СУБД / таблицы продаж за ноябрь–декабрь 2023 гг. и ретроспективные данные за 6 месяцев.

## 📂 Структура и особенности данных
В качестве первоисточника требований и структуры данных использовался официальный [справочник полей](https://disk.yandex.ru/i/3yBmWVHDzgPLbw). Данные разделены на два ключевых слоя:

### 1. Лист `Product` (Подробный срез за ноябрь и декабрь 2023 г.)
Содержит детальную информацию по каждой товарной единице для выявления локальных закономерностей:
* **SKU** — уникальный идентификатор товара.
* **Категория** — широкая товарная группа.
* **Предмет** — наименование предмета (подкатегория/тип товара).
* **Бренд** — бренд-производитель товара.
* **Цена (ноябрь / декабрь)** — средняя цена за продукт в соответствующем месяце.
* **Продажи (ноябрь / декабрь)** — объем продаж (в единицах товара с учетом отказов).
* **Выручка (ноябрь / декабрь)** — объем выручки (₽).
* **Средний % выкупа (ноябрь / декабрь)** — доля выкупленных товаров от общего числа заказанных.
* **Страна** — страна происхождения товара.

### 2. Лист `Dynamic` (Ретроспективные данные за последние полгода)
Собраны в полуавтоматическом режиме для отслеживания макро-трендов и сезонности:
* **Бренд** — бренд-производитель бренда.
* **Месяц** — дата начала отчетного месяца.
* **Продажи** — объем продаж за месяц (в штуках, с учетом отказов).
* **Выручка** — совокупный объем выручки за месяц.
* **Средний % выкупа** — доля выкупленных товаров от числа заказанных за месяц.
